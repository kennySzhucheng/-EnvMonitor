## 整体架构一览

```
                        STM32F103C8T6 核心板
                        (Blue Pill, 插在 H1+H2 排母上)
                                  │
        ┌─────────┬────────┬──────┼──────┬─────────┬─────────┐
        │         │        │      │      │         │         │
      DHT22     OLED    BH1750  HC-SR04  ESP8266   按键×3   LED×3+蜂鸣器
      (J2)      (J3)    (J3共享) (J4)   (J5)     (SW2-4)  (D2-4,LS1)
        │         │        │      │      │         │         │
      1-Wire     I²C      I²C    GPIO   UART     GPIO      GPIO
      (PA0)     (PB6/7)  (同总线) (PB0/1)(PA9/10) (PA4-6)  (PB12-15)
```

**核心设计思路：** STM32 核心板负责"大脑"，PCB 上的外围电路负责"接口"——把所有传感器、按键、显示器的引脚引到核心板的对应 GPIO 上。

## 第1部分：电源电路

```
原理图中的元件: C21(100µF), C22(100µF), 多个 +5V/+3.3V/GND 标签
```

### 电从哪里来？

- Blue Pill 核心板通过 USB 口取 5V 电
- 核心板上的 AMS1117-3.3V 把 5V 稳定成 3.3V 给 STM32 芯片
- PCB 通过排母 H1/H2 从核心板取 +5V 和 +3.3V

### 为什么额外加 C21 和 C22？

```
USB 5V ──┬── C21(100µF) ── GND     ← 滤掉 USB 电源的纹波
          │
          └── 给 HC-SR04 和蜂鸣器供电（它们需要 5V）

核心板 3.3V ──┬── C22(100µF) ── GND  ← 滤掉 3.3V 的纹波
               │
               └── 给其他所有 3.3V 元件供电
```

**大电容 = "蓄水池"**：当某个元件突然需要大电流时（比如 ESP8266 发射 WiFi），电容能瞬间提供电流，防止电压跌落。

### 电源网络统计

| 电源轨 | 电压 | 给谁供电 | 引脚数 |
|--------|------|---------|--------|
| +5V | 5V | HC-SR04(J4)、蜂鸣器(LS1)、1N4148续流(D5) | 8 |
| +3.3V | 3.3V | 所有上拉电阻、DHT22、OLED、BH1750、ESP8266 | 17 |
| GND | 0V | 全部元件的地 | 37 |

## 第2部分：I²C 总线上拉

```
原理图中的元件: R4(4.7k), R5(4.7k), J3(OLED+BH1750 模块接口)
涉及信号: PB6_SCL, PB7_SDA
```

### I²C 为什么需要上拉电阻？

I²C 总线（SCL 时钟线 + SDA 数据线）是一种**开漏输出**接口。设备只能把线路"拉到 GND"（输出低电平），不能主动输出高电平。

```
没有上拉电阻:                   有上拉电阻:
                                 +3.3V
                                   │
3.3V? ← ─ ─ ─ ─ → 悬空!        R4/R5 (4.7k)
                                   │
SCL/SDA 线 ─── 设备拉低=0V       SCL/SDA ─── 设备拉低=0V，不拉=3.3V
```

> **一句话：R4/R5 保证 I²C 线路在不被拉低时保持在 3.3V（高电平）。**

### J3 模块：OLED + BH1750 共用 I²C

```
J3 是 4Pin 排母，插上 OLED 模块或 BH1750 模块：

J3.pin1 ── +3.3V        ← 供电
J3.pin2 ── PB6 (SCL)    ← 时钟线，通过 R4(4.7kΩ) 上拉到 +3.3V
J3.pin3 ── PB7 (SDA)    ← 数据线，通过 R5(4.7kΩ) 上拉到 +3.3V
J3.pin4 ── GND          ← 地
```

OLED 地址 = 0x3C（7位），BH1750 地址 = 0x23（7位）。地址不同，同一条 I²C 总线不会冲突。

## 第3部分：DHT22 温湿度传感器

```
原理图中的元件: R6(4.7k), J2(DHT22 模块接口)
涉及信号: PA0_DHT22
```

### DHT22 的 1-Wire 单总线

DHT22 只用一根数据线就能双向通信——STM32 先发开始信号，DHT22 回复 40bit 数据（湿度16bit + 温度16bit + 校验8bit）。

```
J2 是 4Pin 排母：

J2.pin1 ── +3.3V        ← 供电
J2.pin2 ── PA0(DATA)    ← 数据线，通过 R6(4.7kΩ) 上拉到 +3.3V
J2.pin3 ── NC (不接)    ← 空脚，加 No-Connect 标记
J2.pin4 ── GND          ← 地
```

R6 的作用同样是上拉——保证数据线空闲时保持 3.3V。4.7kΩ 是 DHT22 数据手册推荐的标准值。

## 第4部分：按键电路（本章的重点和难点！）

```
原理图中的元件: SW2/SW3/SW4, R7/R8/R9(10kΩ), C18/C19/C20(100nF)
涉及信号: PA4_KEY1, PA5_KEY2, PA6_KEY3
```

### 按键电路拓扑

以 KEY1 为例，画出电流走向：

```
         +3.3V
           │
        R7 (10kΩ 上拉电阻)
           │
           ├──── [KEY1标签] ────→ H1.pin24 (PA4) → STM32 读取电压
           │
        SW2.pin1 (按键一端)
           ║  按键
        SW2.pin2 (按键另一端)
           │
          GND
           │
        C18 (100nF 消抖电容)
           │
          GND
```

### 这个电路怎么工作？

**按键没按下时：**
- SW2 断开，电流路径：+3.3V → R7 → PA4
- PA4 读到 3.3V = 高电平 = 逻辑 "1"
- C18 通过 R7 充电到 3.3V

**按键按下时：**
- SW2 闭合，电流路径：+3.3V → R7 → SW2 → GND
- PA4 被 SW2 拉到 GND，读到 0V = 低电平 = 逻辑 "0"
- C18 通过 SW2 快速放电

**C18 (100nF) 的作用：**
按键按下/松开的瞬间，金属触点会快速弹跳（机械抖动），产生一串杂乱的脉冲。C18 像一个"小蓄水池"，让电压不能突变，把这些尖刺滤平。配合软件 30ms 延时消抖，达到双重保险。

### ⚠️ 今天踩的坑：开关方向反了！

**错误接法（开关夹在 MCU 和电路之间）：**
```
+3.3V → R → C → SW.pin1 ─║─ SW.pin2 → MCU(PA4)
```
不按时：PA4 悬空（读到随机值）← 绝对错误！
按下时：PA4 读到 3.3V ← 但这也没用，因为不到 0V

**正确接法（MCU 挂在交汇点上，开关控制是否接地）：**
```
+3.3V → R → [R + C + SW.pin1 + MCU] 交汇点
                    │
                 SW.pin2 → GND
```
不按时：PA4 = 3.3V（高）
按下时：PA4 = 0V（低）← 正确！

## 第5部分：LED 指示灯

```
原理图中的元件: D2(红LED), D3(黄LED), D4(绿LED), R10-R12(330Ω)
涉及信号: PB12_LEDR, PB13_LEDY, PB14_LEDG
```

### 为什么 LED 要串电阻？

LED 是电流型器件——电压超过导通电压（红色约1.8V）后，电流会急剧增大烧毁 LED。串一个电阻限制电流：

```
PB12 → R10(330Ω) → D2(红LED) → GND

电流计算: I = (3.3V - 1.8V) / 330Ω ≈ 4.5mA  ← 安全范围
```

| 颜色 | GPIO | 电阻 | 含义 |
|------|------|------|------|
| 🔴 红 | PB12 | R10(330Ω) | 高温报警 |
| 🟡 黄 | PB13 | R11(330Ω) | 湿度异常 |
| 🟢 绿 | PB14 | R12(330Ω) | 人体靠近检测 |

## 第6部分：蜂鸣器驱动电路

```
原理图中的元件: Q1(S8050), R13(1kΩ), R14(10kΩ), D5(1N4148), LS1(蜂鸣器)
涉及信号: PB15_BUZZ
```

### 为什么需要三极管驱动？

STM32 的 GPIO 引脚最大输出电流约 8mA，而蜂鸣器可能需要 30-50mA。三极管就像一个"电子开关"：

```
PB15 (GPIO) ── R13(1kΩ) ── Q1 的 B(基极)
                               │
                        小电流控制 ↓  大电流通过
                               │
                     +5V ── 蜂鸣器 ── Q1的C(集电极) ── Q1的E(发射极) ── GND
```

- GPIO 输出 3.3V → Q1 导通 → 蜂鸣器电流从 +5V 流过 → 蜂鸣器响
- GPIO 输出 0V → Q1 截止 → 蜂鸣器不响

### R13/R14/D5 各自的作用

| 元件 | 作用 |
|------|------|
| R13(1kΩ) | 限制 PB15 → Q1 基极的电流，保护 GPIO 口 |
| R14(10kΩ) | 基极下拉电阻，GPIO 未初始化时保证 Q1 关断（防止一上电乱叫） |
| D5(1N4148) | **续流二极管**——蜂鸣器是感性负载，断开瞬间会产生反向高压，D5 把这个高压短路掉保护 Q1 |

## 第7部分：HC-SR04 超声波测距

```
原理图中的元件: J4(HC-SR04 模块接口)
涉及信号: PB0_HC_TRIG, PB1_HC_ECHO
```

HC-SR04 是 4Pin 模块，用 +5V 供电：

```
J4.pin1 ── +5V           ← 供电 (注意是5V不是3.3V！)
J4.pin2 ── PB0 (TRIG)    ← STM32 发出 10µs 高脉冲触发测量
J4.pin3 ── PB1 (ECHO)    ← HC-SR04 返回高电平，脉宽 = 距离
J4.pin4 ── GND
```

**PB1 需要定时器输入捕获功能（TIM3_CH4）**，这是 CubeMX 中特别配置的，不能用普通 GPIO。

## 第8部分：ESP8266 WiFi 模块

```
原理图中的元件: J5(ESP8266-01S 模块接口)
涉及信号: PA9_TX, PA10_RX, ESP_RST, ESP_GPIO0
```

ESP8266-01S 是 2x4Pin（8脚）模块：

| 脚号 | 信号 | 连接 | 说明 |
|------|------|------|------|
| 1 | VCC | +3.3V | ⚠️ 必须 3.3V，不能接 5V！ |
| 2 | GPIO2 | NC | 内部已上拉，不接 |
| 3 | CH_PD/EN | +3.3V | 使能脚，必须拉高模块才工作 |
| 4 | RST | H2.p9 | 复位脚 |
| 5 | TX | PA10 (USART1_RX) | ESP 发送 → STM32 接收 |
| 6 | RX | PA9 (USART1_TX) | STM32 发送 → ESP 接收 |
| 7 | GPIO0 | NC | 烧录固件时才用到 |
| 8 | GND | GND | 地 |

> TX/RX 是交叉连接的：A 的 TX 接 B 的 RX，A 的 RX 接 B 的 TX。很多人第一次都会接反！

## 原理图检查清单

在进入 PCB 设计前，确认以下每一条：

- [ ] 每个元件的 Value 正确（电阻阻值、电容容值）
- [ ] 每条信号标签都放在导线上
- [ ] 所有电容负极接地（不是 No-Connect）
- [ ] 大电容 C21/C22 正极接电源、负极接地
- [ ] ESP8266 供电是 +3.3V 不是 +5V
- [ ] HC-SR04 供电是 +5V
- [ ] I²C 有上拉电阻 R4/R5 (4.7k)
- [ ] DHT22 有上拉电阻 R6 (4.7k)
- [ ] 按键电路：R+C+SW+MCU 在同一交汇点，SW 另一端接地
- [ ] ERC 错误全部分析过（不是简单清零）
- [ ] 每个元件都分配了正确的封装 (Footprint)
