# 🔧 KiCad 入门讲解 — 从零到第一个原理图

> 本教程基于 EnvMonitor 项目的实际经验编写，适合零基础新手。

---

## 1. KiCad 是什么？

KiCad 是一套**免费开源**的电路设计软件，用来画电路原理图和设计 PCB 电路板。

```
你的想法 → 画原理图（这个元件连那个元件） → 画PCB（铜线怎么走） → 导出文件发工厂 → 拿到实物板子
```

和立创EDA相比：KiCad 完全免费无限制、文件格式开放、全球开源社区支持。缺点是学习曲线稍陡。

---

## 2. KiCad 的界面组成

打开 KiCad 后会看到主界面，里面有几个核心工具：

| 图标 | 工具 | 干什么用 |
|------|------|---------|
| 📝 | **原理图编辑器 (Eeschema)** | 画电路图，放元件，连线 |
| 🟢 | **PCB 编辑器 (Pcbnew)** | 设计电路板的铜线走线 |
| 📦 | **封装编辑器** | 编辑元件的物理尺寸（焊盘位置） |
| 🔣 | **符号编辑器** | 编辑元件的原理图符号（那个小方块图） |

> 新手只需要关注前两个：原理图编辑器 + PCB 编辑器。本讲解聚焦原理图编辑器。

---

## 3. 原理图编辑器的核心概念

### 3.1 元件（Symbol） — 电路里的"零件"

每个元件就是一个符号，比如电阻画成一个锯齿、电容画成两条平行线。元件上伸出小线头叫**引脚（Pin）**，连线就接在引脚上。

### 3.2 导线（Wire） — 电流的"路"

用导线把两个元件的引脚连起来，电流就能通过了。快捷键 `W`。

### 3.3 标签（Label） — "无线导线"

**这是 KiCad 中最重要也最容易出错的概念！**

```
如果你有两段导线离得很远，画线太乱，怎么办？
→ 在每段导线上各放一个同名标签，KiCad 会把它们当成同一条网络！
```

**坑点：标签必须放在导线上才能生效。** 放在空白处的标签是"死标签"——它创建了一条孤立的网，不和任何东西连通。你今天遇到的 KEY1/KEY2/KEY3 悬空问题就是这个原因。

### 3.4 网络（Net） — "同一条电路"

所有通过导线或同名标签连在一起的引脚，都属于**同一条网络**。比如 `KEY1` 网络包含 R7.pin2 + SW2.pin1 + C18.pin1 + H1.pin24，这四个点电压始终相等。

### 3.5 电源符号（Power Symbol） — 特殊的"全局标签"

`+3.3V`、`+5V`、`GND` 这些是特殊的全局标签。**只要原理图上出现同名的电源符号，它们自动连通，不需要画线。**

所以你可以到处放 GND 符号——它们全都连通。

### 3.6 No-Connect 标记（×号） — "这个脚故意不接"

有些元件的某些脚你真的不需要用。如果不加任何东西，ERC 会报"引脚未连接"。放一个 × 号就是告诉 KiCad："我知道这个脚空着，别报警。" 快捷键 `Q`。

### 3.7 PWR_FLAG — "电从这里来"

KiCad 需要知道每条电源轨的"电源从哪里输入"。PWR_FLAG 就是一个标记，告诉你："+5V 由核心板提供，不是漏画了电源芯片。"

---

## 4. ERC（电气规则检查）— 原理图的"编译"

**ERC = Electrical Rules Check = 检查你的电路有没有低级错误。**

就像写代码后的编译——程序"语法正确"不代表逻辑正确。ERC 能发现：
- 引脚忘了连线
- 标签悬空
- 电源网络没有电源来源
- 两个输出脚短路在一起

**但是：ERC 清零 ≠ 电路正确！** 你今天遇到的电容 No-Connect 问题就是例子——ERC 不报错，但消抖电容不接地就不工作。

## 5. 常用快捷键速查

| 快捷键 | 功能 |
|--------|------|
| `W` | 画导线 |
| `L` | 放置标签 |
| `P` | 放置电源符号 |
| `Q` | 放置 No-Connect 标记 |
| `M` | 移动选中元件 |
| `G` | 拖动元件（导线跟着走） |
| `R` | 旋转元件 |
| `Delete` | 删除选中项 |
| `Ctrl+Z` | 撤销 |
| `Ctrl+S` | 保存 |

## 6. 一个完整的按键电路是怎么画出来的？

以 KEY1（PA4 按键）为例：

```
步骤1: 从库中拿出 SW2（按键开关）、R7（10k电阻）、C18（100nF电容）
步骤2: 摆放它们的位置，让 R7 在上面，SW2 在中间，C18 在下面
步骤3: 用 W 键画线：
       R7 上端 → +3.3V 电源符号
       R7 下端 + SW2 一脚 + C18 上端 → 三个连在同一个点
       SW2 另一脚 → GND
       C18 下端 → GND
步骤4: 在 R7+SW2+C18 的交汇导线上按 L，输入 "KEY1"，放置标签
步骤5: 在 H1 核心板插座旁边放一个 KEY1 标签 → 自动与上一步的 KEY1 合并
```

> 这就是标签的妙用——不用从按键画一根长线横跨整个原理图到核心板插座，放两个同名标签就完成了！

---

## 7. 新手常犯错误 Top 5

| 排名 | 错误 | 怎么发现 |
|------|------|---------|
| 1 | 标签放在空白处 | ERC 报 "label_dangling" |
| 2 | 电阻电容在开关两边 | 手动画出电流路径检查 |
| 3 | 电容负极用 No-Connect 而不是 GND | ERC 不会报错，但电路不工作 |
| 4 | 开关夹在MCU和交汇点之间 | MCU 引脚悬空，读到随机电压 |
| 5 | 保存了原理图但没更新 ERC 导出 | 对着旧报告改了半天白改 |

## 8. 下一步：从原理图到 PCB

原理图完成后，下一步是 PCB 设计：

1. **给每个元件分配封装（Footprint）** — 告诉 KiCad 每个元件的实物尺寸
2. **导入到 PCB 编辑器** — 所有元件带着飞线（白色细线）出现在板子上
3. **摆放元件** — 像拼图一样把元件放在板上合适的位置
4. **布线** — 按照飞线的指引，画铜线连接各个引脚
5. **铺铜 + DRC 检查** — 在空白处铺上大面积的 GND 铜皮，运行设计规则检查
6. **导出 Gerber 文件** — 发给嘉立创打样
